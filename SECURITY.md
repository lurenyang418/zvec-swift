# Security Policy

## Supported versions

Security updates are provided for the latest tagged release.

## Reporting a vulnerability

Do not open a public issue. Use [GitHub private vulnerability reporting](https://github.com/lurenyang418/zvec-swift/security/advisories/new). Include affected versions, reproduction steps, impact, and any proposed remediation. Reports are acknowledged within seven days.

Issues in the underlying engine should also be reported to the [Alibaba Zvec security process](https://github.com/alibaba/zvec/security).
